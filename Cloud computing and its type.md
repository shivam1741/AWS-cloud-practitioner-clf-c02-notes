# Cloud Computing and its types
## What is Cloud Computing 
* Cloud computing means using computing services over the internet instead of your own physical computer or server. These services include servers, storage, databases, networking, and applications.
* Instead of buying and maintaining expensive hardware, companies rent these resources from cloud providers like Amazon Web Services, Microsoft (Azure), or Google (Google Cloud). The infrastructure is stored in large data centers, and users access it through the internet whenever they need it.
* For example, if a company wants to launch a website, it does not need to buy its own server. It can create a virtual server on a cloud platform such as Amazon EC2, upload the website files, and make the site accessible to people around the world. When users open the website, their request travels through the internet to the cloud server, the application processes the request, and the result is sent back to the user’s browser.
* Cloud computing is widely used because it allows businesses to scale resources easily. If more users visit a website, the company can quickly increase server capacity without buying new hardware. If traffic decreases, it can reduce resources and save money.

## Cloud Computing types
* Cloud computing is divided on the basis of the level of service provided by the cloud provider and the level of control or responsibility of the user.
* In other words, it depends on how much of the infrastructure, platform, and software is managed by the cloud provider and how much is managed by the user.

### 1. IaaS (Infrastructure as a Service)
In **IaaS**, the cloud provider gives you **virtual machines, storage, and networking**, and you manage the rest.

You control:

* Operating system
* Applications
* Security configurations

The provider only manages the **physical hardware and virtualization**.

Example service: **Amazon EC2** from **Amazon Web Services**.

Example use:

* Creating virtual servers
* Building custom networks
* Hosting applications manually


### 2️. PaaS (Platform as a Service)

In **PaaS**, the cloud provider manages **servers, operating systems, and runtime environments**, so developers only focus on **writing and deploying code**.

You control:

* Application code
* Data

The provider manages:

* OS
* servers
* scaling

Example service: **AWS Elastic Beanstalk**.

Example use:

* Developing web applications
* Deploying apps without managing servers


### 3. SaaS (Software as a Service)

In **SaaS**, the complete software is provided through the internet. Users just **open the application in a browser**.

You control almost nothing; everything is managed by the provider.

Examples:

* **Gmail**
* **Google Docs**
* **Dropbox**

Example use:

* Email services
* online document editing
* file storage
